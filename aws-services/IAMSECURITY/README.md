# CONCEITOS BÁSICOS DE SEGURANÇA (AWS IAM)

Gerenciamento de identidades e controle de acesso aos recursos da AWS utilizando o princípio do menor privilégio.

## Conceitos Principais
- IAM (Identity and Access Management)
- Princípio do menor privilégio
- Grupos e usuários
- Políticas gerenciadas
- Segurança em nuvem

## Estrutura
- IAM Users
- IAM Groups
- IAM Policies
- Políticas gerenciadas pela AWS

## Implementação
- Criação de grupos IAM
- Criação e associação de usuários
- Aplicação de políticas de acesso somente leitura
- Controle de permissões para EC2 e RDS

## Casos de Uso
- Controle de acesso corporativo
- Gerenciamento de equipes técnicas
- Ambientes de produção seguros
- Separação de responsabilidades

## Boas Práticas
- Evitar uso do usuário root
- Aplicar permissões mínimas necessárias
- Usar grupos para gerenciar acessos
- Revisar permissões periodicamente
